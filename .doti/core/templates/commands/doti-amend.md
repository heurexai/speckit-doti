# doti-amend

Purpose: amend an already-stamped cycle stage after an approved change to its artifact (a spec/plan/tasks edit, or a follow-up code change), and reconcile the cycle state HONESTLY â€” re-establish freshness through the recovery plan instead of hand-guessing which stamps still hold. A utility: it runs inside the active feature cycle and never reorders `/01`â€“`/09`.

Command-backed behavior:

1. Read `.doti/agent-context.md`.
2. Classify what actually changed: `hx doti review-context --repo .` projects the current change set into change categories and which arch-review lenses it touches. Use it to scope the amendment â€” a docs/prose-only edit reaches different stages than a production-code edit.
3. Derive the recovery plan, never a guess: `hx doti cycle refresh-plan --target <stage>` reports, per transitively-required stage, whether its existing stamp is a `SafeReinterpret` (binding-format migration â€” re-bindable), `ReBindContentEqual` (a content-equal / edge-only prerequisite lag â€” the stage's OWN artifact is unchanged under canonical hashing, every producing upstream is fresh, and every shared prerequisite's content is byte-identical, so only the binding edge moved â€” auto-rebindable; a real content change can NEVER reach this), `ReviewedNoImpact` (an upstream artifact's content genuinely changed and this non-review, non-code-bound stage now needs an explicit agent IMPACT DECISION â€” the engine surfaces the changed upstream paths + the line-level diff on the recovery step; you either re-author the stage if the change affects it, or record a no-impact attestation if it does not; a bare `stamp` REFUSES until you do one of those), `RerunRequired` (the artifact's content genuinely changed â€” the stage must be re-run through its `/0N` skill; review-kind and change-set-bound stages are always here), `inserted-stage` (the current graph requires a stage cycle-state never stamped â€” run its named `/NN` skill to produce + stamp it), or `NotBound`. This is the same freshness logic `doti cycle check` enforces, surfaced as a plan.
4. Apply the safe re-bindings: `hx doti cycle refresh --target <stage> --apply-safe` re-stamps both `SafeReinterpret` AND `ReBindContentEqual` steps (re-deriving the plan after each, so a content-equal chain settles in one pass). It NEVER fabricates a stamp for a `RerunRequired`/`inserted-stage` step â€” re-run that stage's `/0N` skill so its proof is earned. **You usually do not need this step at all:** re-running the one genuinely-changed stage now AUTO-CASCADES the content-equal rebind to its content-unchanged dependents â€” the old stage-by-stage hand-re-stamp treadmill is gone. A genuine content change, a review-kind stage, or a code-bound stage still earns a real re-run.

   For a `ReviewedNoImpact` step, the engine has done the bookkeeping â€” READ the surfaced upstream diff (the recovery step carries the changed prerequisite paths + the `git diff` since this stage was last validated), reason about whether those changes affect THIS stage's artifact, then do exactly one: re-run the stage's `/0N` skill (the change has impact â€” re-author it), OR record an explicit no-impact verdict with `hx doti cycle review-rebind --target <stage> --attest no-impact [--reason "<why>"]` (you reviewed the diff and it does not affect this stage). Re-stamping to clear the flag WITHOUT assessing the diff is a rubber-stamp and is forbidden â€” a bare `hx doti cycle stamp` of such a stage REFUSES (`Validation_CycleReviewRebindRequiresAttest`) and routes you to the verb. The attestation is recorded (dependent stage, the upstream that changed, beforeâ†’after hashes, your reason) and decays automatically if that upstream later moves again.
5. Confirm the cycle is whole: `hx doti cycle check` must pass (every transitive prerequisite stamped + fresh) before you continue. amend reconciles within the cycle; it does not invent a new stage or change the `/01`â€“`/09` order.

Expected output: the review-context classification, the recovery plan (per-stage `SafeReinterpret`/`RerunRequired`/`NotBound`), the stages re-bound by `--apply-safe`, and the list of stages you re-ran by hand â€” ending with a green `doti cycle check`.
