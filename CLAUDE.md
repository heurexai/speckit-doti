# Claude Entry Point

This file is a command-aware-advisory entrypoint for Claude.

Read `.doti/agent-context.md` first. Use local Claude skills under `.claude/skills/*doti-*`.

`Hx.Scaffold.Cli new` is the single generation entry point and runs trusted prerequisite preflight before output mutation; operational `hx` commands require an executable-adjacent `hx.config.json`; `hx prereq check` reports .NET SDK/Git/directory readiness and `hx prereq install` can run only a Windows winget plan whose release-defined package/source metadata and manifest identity match an exact operator-approved digest. `hx release` is vendored-target-aware and Velopack-backed: it validates the executable-local Microsoft Configuration file, validates `.doti/release.json`, verifies the GitVersion-calculated major/minor/patch intent, creates or verifies the local annotated release tag, publishes the repo's declared executable, stages vendored assets into the Velopack pack directory, runs `vpk pack`, records the target, tag, payload checks, config source, and package artifacts in `LocalReleaseResult`/`release.identity.json`, and copies artifacts to the configured local release root when `localReleaseOutput.enabled` is true. `hx release` does not push tags; `/09-doti-release` owns the later remote push/CI verification. `doti cycle stamp --stage release --release-intent <major|minor|patch>` records the release intent in the automatic drift-review transition commit as a GitVersion `+semver:` signal before `hx release` validates the calculated version. `gate run --profile <auto|advisory|normal|release>` aggregates the deterministic ladder into one fail-closed `GateProof`; the proof includes canonical affected-test planner, test-scope, and execution hashes that Doti workflow transitions and release paths recompute before relying on them. The affected-test planner (`Hx.Impact.Cli plan`) deterministically scopes the gate's normal/advisory test run (project-graph reverse-closure -> covering test projects; fail-closed escalation); release runs the full suite. The self-hosted gate executes prebuilt test outputs after the command-backed build so it cannot overwrite its own loaded runner assemblies. Human help for all scaffold CLIs routes through one shared rich/plain renderer; `--help-mode plain`, `--plain-help`, `HX_HELP_MODE=plain`, and `NO_COLOR` provide ANSI-free help. Versioning is GitVersion-backed (`version calculate`; release tag creation lives in `hx release`). The security gate (`security scan`) runs package-vulnerability SCA (`dotnet list package --vulnerable`) + reports the build-integrated analyzer SAST status (CA3xxx/CA5xxx as errors — the build is the SAST enforcement point); enforced at release, advisory in dev. The .NET analyzers + Gitleaks are the SAST gate. Vendored tools (Gitleaks/Sentrux/GitVersion) are pinned + hashed; Sentrux is the Heurex fork v0.5.11 for declared RIDs with matching grammars, and large binaries are gitignored as an operational vendor step. The gate never creates a Sentrux baseline. Installed skills are rendered from one source — edit `.doti/core/skills.json` and re-render; do not hand-edit installed skill files. The doti cycle-state substrate (`doti cycle stamp`/`status`) records diff-bound stage proofs + freshness; `stamp` refuses a first-stage feature slug that is not numbered (`NNN-short-name`) and refuses non-initial stages until every transitive prerequisite is stamped + fresh. The enforcing chokepoints are `doti cycle check` (every transitive prerequisite stamped + fresh), `doti question check` (the operator-question validator), plus an untracked, logic-free insurance pre-commit hook (`doti install-hooks`) that blocks bare commits. `Hx.Scaffold.Cli new` and `doti install` auto-arm/refresh the hook for Git repos and fail hard rather than overwrite non-Doti hooks. `gate run` persists a change-set-bound proof. The workflow is command-enforced at the chokepoints, and commits are owned by coded Doti workflow transitions and release paths.
