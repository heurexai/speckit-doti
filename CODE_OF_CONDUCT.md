# Code of Conduct

## Our pledge

We want speckit-doti to be a focused, respectful place for people building deterministic .NET scaffolds and agent-driven workflows.

Contributors and maintainers are expected to:

- be respectful and constructive;
- assume good intent while still being precise about technical problems;
- keep criticism focused on code, design, behavior, and evidence;
- make space for different levels of experience;
- avoid harassment, discrimination, personal attacks, and unwelcome attention.

## Unacceptable behavior

Unacceptable behavior includes:

- insulting, demeaning, or threatening language;
- harassment or discrimination of any kind;
- publishing someone else's private information;
- sexualized language or imagery in project spaces;
- repeated disruption of issues, pull requests, or maintainer decisions.

## Enforcement

Maintainers may remove comments, close issues or pull requests, restrict participation, or block accounts when behavior is inconsistent with this code of conduct.

If you need to report a conduct issue, contact a maintainer through an available private maintainer channel. If no private channel is available yet, open a minimal public issue asking for a private maintainer contact path and do not include sensitive personal details.
