# Tasks: Standalone, Source-Free Hx CLI and Distribution

Plan: `docs/plans/007-standalone-hx-source-independent-cli-plan.md`. Tasks are organized into **mandatory sequential phases (Phase 0 → Phase 7)** and numbered **T001 → T047 in execution order**. Each `FR-###`/`SC-###` maps to at least one task; a test task precedes the implementation it covers.

## Execution order — phases are SEQUENTIAL (mandatory)

**Implement strictly top-to-bottom, by phase and by task number. Work only the lowest-numbered incomplete task. Do NOT jump to a later phase, and do NOT pick tasks by topic.** This order is dependency-driven: the framing + substrate (Phases 0–1) must exist before the distribution (Phase 4) ships the target structure — building out of order produces throwaway/retrofit.

A phase is "done" only when its tasks are checked **and** `gate run` (T044) is green; the next phase starts only then. If a task is blocked, surface it — do not skip ahead. (T003 makes this a fail-closed machine gate so it cannot be ignored.)
