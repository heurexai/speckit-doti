# Tasks: Standalone, Source-Free Hx CLI and Distribution

Plan: `docs/plans/007-standalone-hx-source-independent-cli-plan.md`. Tasks are organized into **mandatory sequential phases (Phase 0 ‚Üí Phase 7)** and numbered **T001 ‚Üí T047 in execution order**. Each `FR-###`/`SC-###` maps to at least one task. **Test-first holds in the distribution + reconciliation phases (Phases 2‚Äì5: T009‚ÜíT010/T011, T013‚ÜíT014, T019‚ÜíT020, T027‚ÜíT028, T029‚ÜíT030);** the Phase-0 substrate (T002/T003) and Phase-6 absorptions (T033‚ÄìT041) land with their tests **inline in the same task** (each carries an explicit test requirement ‚Äî see the task text), not as a separate preceding task, to avoid renumbering the ordered-task chain. The arch-review record is `docs/reviews/007-standalone-hx-source-independent-cli-arch-review.md`.

## Execution order ‚Äî phases are SEQUENTIAL (mandatory)

**Implement strictly top-to-bottom, by phase and by task number. Work only the lowest-numbered incomplete task. Do NOT jump to a later phase, and do NOT pick tasks by topic.** This order is a deliberate capture-sequencing choice (framing + substrate first so the distribution lands on a stable enforcement floor and nothing is retrofitted). **Honesty note (arch-review):** the distribution core (Phases 2‚Äì5) has no *code* dependency on the substrate (Phases 0‚Äì1) or the absorptions (Phase 6) ‚Äî the arch-review confirmed the dependency runs the other way (the tier model consumes the installed `.doti` payload the distribution produces). So a later **implement-time split** into Feature A (distribution, Phases 2‚Äì5 + distribution slices of 7), Feature B (substrate, Phases 0‚Äì1), Feature C (absorptions, Phase 6) is supported and **preserves this phase order**; until then, keep T003's ordered-task gate scoped so a snag in the speculative Phase 0 does not wedge the urgent distribution fix.

A phase is "done" only when its tasks are checked **and** `gate run` (T044) is green; the next phase starts only then. If a task is blocked, surface it ‚Äî do not skip ahead. (T003 makes this a fail-closed machine gate so it cannot be ignored.)
