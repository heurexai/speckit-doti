## Research (resolve unknowns)

No open unknowns.

- **Decision:** split the encoding into usage (agent context, cross-cutting), design (plan template + `doti-plan`), and execution (`doti-implement`).
- **Rationale:** the usage directive must apply at *every* stage — especially implement/debug, where reading a diagnostic `code`/`hint` matters most — so it belongs in the always-loaded agent context, not only the plan.
- **Alternatives rejected:** plan-only (too narrow — the agent wouldn't self-correct at implement time); agent-context-only (would miss *designing* new commands to be self-describing).

## Design

Edit (source only), then re-render:

1. `doti/core/templates/agent-context-template.md` — add a Workflow-Rules directive: run `describe --json` to learn a scaffold CLI before driving it; on any non-Success `CliResult`, act on the diagnostics `code` / `hint` / `nextActions` instead of guessing or blind-retrying.
2. `doti/core/templates/plan-template.md` — add a `## CLI surface & error contract` section: emitted error codes (→ `errorcodes/registry.json`), exit class, `describe` entry, `CliResult` envelope conformance; omit when the feature adds no CLI surface.
3. `doti/core/templates/commands/doti-plan.md` — add a Behavior step that produces that section.
4. `doti/core/templates/commands/doti-implement.md` — add an execution step: register declared codes in `errorcodes/registry.json`, run `errorcodes check` (append-only stability gate), and confirm `describe` reflects the new surface.
5. `doti render-skills --repo . --agents codex,claude` — regenerate `.doti/agent-context.md` + the skill files.

**Architecture delta:** none. No projects / namespaces / layers added or moved; `rules/architecture.json` and `.sentrux/rules.toml` unchanged.

**CLI surface & error contract (this feature):** none — it edits workflow templates only and adds no command, so the new plan-template section does not apply to this feature's own plan.
