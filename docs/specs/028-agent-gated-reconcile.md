# 028 — Self-describing, agent-gated cycle reconciliation

## Summary

027 made the engine refuse to silently rebind a stage when an upstream artifact's content changed (a genuine content change is always `RerunRequired`, never auto-rebound). That closed the *machine* rubber-stamp hole. It left the *agent* rubber-stamp hole open: an orchestrating agent that sees `RerunRequired`, re-runs the stage perfunctorily, and re-stamps to clear the flag without actually checking whether the upstream change affected this stage.

028 closes that hole by making the workflow **self-describing**: the engine does the bookkeeping and hands the agent the exact upstream diff (which file, which lines) a decision needs AND the valid next-action commands; the agent reasons over information already laid out and records an explicit, fenced verdict. The engine then propagates that verdict mechanically — rebinding the one stage the agent attested is unaffected, requiring a real re-author where the agent found impact, and surfacing each further-downstream stage for its *own* decision.

This is an explicit state machine over the existing stage chain (a last-completed-step marker + a content-addressed dependency graph), extended by one new, fail-closed decision — an **agent-gated reviewed-no-impact rebind**, fenced *inside* `Stamp` and recorded — and by a single code-level **action model** from which the workflow's "what can I do next" affordances are generated, so the agent never has to recall a command.

## Motivation

The engine can *force* a re-run; it cannot *force* genuine reasoning. The only way to make the forced re-run substantive is to (a) give the agent the precise evidence + its valid choices, and (b) let the agent's informed, recorded judgment — not a blind re-stamp — drive the rebind. This is a deliberate, conscious trust model: from "engine-prevented rubber-stamp" (027) to "engine-*informed* and recorded judgment" (028). The fences keep it from collapsing back into a rubber-stamp, and the action model keeps the affordances from drifting.

## Functional requirements

- **FR-002 — Surface the upstream diff (self-describing staleness), at the recovery/CLI seam.** When a stage is stale via `PrereqArtifactChanged`, the engine surfaces the precise delta: the changed prerequisite path(s) — exactly the keys where the bound vs current prerequisite content hashes differ — carried on the `StageRecoveryStep`, plus a line-level `git diff` of those paths since the stage's stamp commit. The path-set comes from the pure `FreshnessEvaluator`/`CycleRecoveryPlanner` (no git there); the **`git diff` runs at the CLI / recovery-presentation seam** (where process execution already lives), computed **lazily** (only when an attest-eligible step's diff is requested). When `StampedAtCommit` is null (unborn/detached HEAD), fall back to a worktree diff against the bound content and label it; never throw.

- **FR-003 — Reconciliation mechanism: content-binding rebind via a dedicated attestation verb that evaluates its target directly.** The rebind MECHANISM is the existing content-binding re-stamp (rebinds `PrerequisiteArtifactHashes` to current), so decay and atomicity come for free. The no-impact DECISION is a dedicated verb — `doti cycle review-rebind --target <stage> --attest no-impact [--reason <text>]`, never a bare `Stamp`. The verb reads the **target's own** freshness directly via `FreshnessEvaluator.Evaluate(targetProof, requireChangeSetIdentity)` (the primitive `Status()` already uses) — NOT `Check(target)`, which evaluates only the target's prerequisites and never the target's own `PrereqArtifactChanged`.

- **FR-004 — Eligibility fence INSIDE `Stamp` (one pure compound predicate, fail-closed).** The compound predicate — `PrereqArtifactChanged` AND non-review-kind AND `!requireChangeSetIdentity` — is a single pure function (`ReviewRebindEligibility.IsAttestable(StageFreshnessResult, CycleStage)`), unit-testable git-free. It is enforced **inside `CycleService.Stamp`**, not beside it: a bare `Stamp` that would clear an *attestable* stale (own artifact unchanged, only prerequisite hashes diverge) **refuses** — throwing a `CycleInputException` routed to `Validation_CycleReviewRebindRequiresAttest`, directing the agent to `review-rebind --attest no-impact`. A real re-author (own-artifact hash changed) stamps normally. `Refresh --apply-safe` continues to apply only `SafeReinterpret`/`ReBindContentEqual` (never the new `ReviewedNoImpact` tier). So no path — `stamp`, `refresh`, or the cascade — can clear an attestable stale without the recorded verdict. `OwnArtifactChanged`/`ChangeSetDiffers`/`NotProduced`/review-kind/change-set-bound stay hard `RerunRequired`.

- **FR-005 — Auditable record in a dedicated field; one atomic write; cascade suppressed.** A reviewed rebind writes a distinct `CycleStageOutcome.ReviewedNoImpactRebound` proof outcome plus an immutable `CycleReviewedRebindRecord` (dependent stage, changed upstream stage(s), **before→after** prerequisite hashes, verdict, optional reason) to a **new additive nullable field `CycleState.ReviewedRebinds`** (mirroring `CompletedUnreleasedCycles`/`ReleasedCycles` — NOT `Transitions`, which is typed `IReadOnlyList<CycleTransitionRecord>` and scanned by the release-train). The before→after hashes are assembled from values already in hand (the existing proof's `PrerequisiteArtifactHashes` vs the freshly-computed `currentPrereqs`) — no new content-comparison path. The rebound state (proof + record) is built and persisted in **one** `CycleStateStore.Write` (atomic by the single whole-state write — SC-007 reframed to "the rebound state is one atomic write"). The verb **suppresses** `CascadeSafeRebindAfterStamp` for the attested rebind: it rebinds ONLY the target; each further-downstream stage surfaces for its own attestation, so one record never silently flips multiple stages Fresh.

- **FR-006 — Diff-carrying propagation (one worklist, target-only rebind).** The existing `Refresh --apply-safe` fixpoint is unchanged for `SafeReinterpret`/`ReBindContentEqual`; `RerunRequired` emits a re-author step carrying the diff payload (FR-002); an attested no-impact rebind is applied only by the verb, to the target only (cascade suppressed). A reviewed-rebind is not a forward transition: it refreshes a binding in place; it does not move the marker or commit.

- **FR-007 — Last-step marker + single-sourced chain.** Keep `CurrentStage` as the explicit last-*authored*-step marker. Single-source the graph on `StageModel`: de-duplicate the `CheckHelpers` transitive-prereq DFS into `StageModel.TransitivePrereqStages`; **add a `Next` field to `CycleStage` + the `StageEntry` binder and declare `next:` on all 9 stages in `workflow.yml`** (today only `drift-review` declares it and the binder ignores it — 8/9 successor edges are a synthesized fallback). Additive within schemaVersion 2 (`IgnoreUnmatchedProperties`).

- **FR-008 — Anti-rubber-stamp agent prompts.** The re-runnable `/0N` templates + `/doti-amend` + `/doti-drift-fix` gain the reminder: on a re-run caused by an upstream change, read the surfaced diff, reason about impact, then either re-author OR record a reviewed-no-impact verdict — clearing the flag without assessing impact is forbidden. Single-sourced + re-rendered.

- **FR-009 — Lock the reconciliation matrix (carried from 027).** Pin the two un-asserted cells: a change-set-bound stage on a doc-only upstream edit → `PrereqArtifactChanged` → `RerunRequired`; the spec+plan union.

- **FR-010 — Self-describing WORKFLOW affordances, generated from one code model (no drift).** The agent's valid **workflow** next-action commands — stage advance, the recovery menu, the `attest no-impact` verb, the publish-boundary STOP, the train-loop, the bug-phase steps, and the utility skills — are GENERATED from a single `DotiActionModel` registry of `CommandDescriptor`s. Each declares an `InvocationTemplate` (the one source of exact flags) + **declarative applicability** (`When`) over a closed named-condition vocabulary (`StageCurrent`/`CheckPassed`/`RecoveryTier`/`GateFailed`/`TrainValid`/`OutsideCycle`/`BugPhase`, `All`/`Any`) — not an opaque predicate — so the engine evaluates applicability AND renders the "available when …" from the same data. Stage identity (ordinal, `/0N`, produces, prereqs, `next`) projects from `StageModel`; the **recovery-menu descriptors WRAP the existing single `CycleRecoveryPlanner` projection** (tag each `StageRecoveryStep` with its descriptor id — never a second evaluator that could disagree, the 027 hazard). A pure `DotiActionProjector.Project(CommandContext)` yields the valid actions, projected onto: `CliResult.NextActions` (workflow results), the stale-recovery menu, the rendered skill refs + the agent-context `{commandAvailability}` placeholder + `hx describe` (via an explicit, reviewed `Hx.Doti.Core → Hx.Cycle.Core` project edge; acyclic), with the descriptor→`CliNextAction` mapping in `Hx.Runner.Cli` (keeping `CliResults`/the kernel domain-agnostic). It **subsumes `DotiWorkflowRegistry`** — including its presentation data (stage status/title/next-step) which is rendered from the descriptors + `StageModel`. The 7 **utility skills** (doti-bug/amend/converge/drift-fix/constitution/auto/upgrade) are `Utility`-kind descriptors (so deleting `skills.json nextStage` does not strand them). **Scope boundary:** payload-derived next-actions that are functions of a computed result, not cycle state — `ImpactCommands` affected-test guidance, `Scaffold.Cli` per-prereq install hints, install hook paths — stay locally constructed; the "no hand-authored next-action" guard is scoped to the workflow-affordance class via an explicit allow-list. Registry static-invariants: one enabled advance per stage; no two descriptors apply at one decision point; no decision point yields zero actions. ACID test (for the affordance class): change a flag/applicability in one descriptor → every workflow surface updates; nothing to hand-edit.

## Success criteria

- **SC-001** — Edit the spec, surface the diff, attest no-impact on `plan` → `plan` is rebound (`Fresh`) with a recorded `CycleReviewedRebindRecord` in `ReviewedRebinds`; a later spec edit re-stales `plan` (decay).
- **SC-002** — The pure `IsAttestable` predicate returns false for `OwnArtifactChanged`/`ChangeSetDiffers`/`NotProduced`/review-kind/change-set-bound (git-free matrix test); a bare `doti cycle stamp` on an attestable stale **throws** `Validation_CycleReviewRebindRequiresAttest`; `Refresh --apply-safe` never applies `ReviewedNoImpact`.
- **SC-003** — After a no-impact rebind, an unrelated transition (`RebaseProofsToHead`) does not break decay: a later upstream move still re-stales (ordinary content-bound proof, no snapshot).
- **SC-004** — The recovery step lists exactly the changed prerequisite paths; the line-level diff renders at the CLI/recovery seam, lazily, and is null-`StampedAtCommit`-safe.
- **SC-005** — The §1 invariant is present + rendered; `render-skills --check` + `payload check` clean.
- **SC-006** — The `/0N` + amend/drift-fix prompts carry the anti-rubber-stamp reminder (rendered, parity clean).
- **SC-007** — The rebound state (proof + `ReviewedRebinds` entry) is built and persisted in one `CycleStateStore.Write` (no partial/rebound-without-record state is representable).
- **SC-008** — Gate green (normal + release); the FR-009 matrix cells locked.
- **SC-009** — Every WORKFLOW decision point's valid commands are generated from `DotiActionModel` and surfaced on the result/skill the agent already reads; the `attest no-impact` choice is presented with its exact invocation at, and only at, a fence-eligible stale; the recovery menu is the single `CycleRecoveryPlanner` projection (no second evaluator).
- **SC-010** — Zero hand-authored **workflow** command info remains: the workflow-affordance `CliNextAction` sites + the skill/agent-context command refs render from the model; `skills.json nextStage` deleted (utility skills covered by descriptors); changing one descriptor updates `nextActions`, skills, agent-context, and `describe` together (single-edit drift test). Payload-templated next-actions are explicitly allow-listed, not authored ad-hoc.
- **SC-011** — Each command's "available when …" renders from its declarative `When`; the registry static-invariants pass; the affordance-scoped `new CliNextAction` guard passes.

## Design direction (for `/04-arch-review` — completed; resolutions folded in)

> Content-addressed dependency DAG + demand-driven dirty-propagation + Salsa-style early-cutoff + event-sourced log, extended by one agent-gated reviewed-no-impact rebind (fenced inside `Stamp`, recorded in `ReviewedRebinds`); workflow affordances projected from a single `DotiActionModel` that wraps the existing `CycleRecoveryPlanner` projection.

Reconciliation = **HYBRID** (content-binding rebind + the verb + the in-`Stamp` fence + the derived `ReviewedRebinds` record; no snapshot). Action model = **GENERATE, scoped to the workflow-affordance class**, declarative applicability, single projection. See the plan for module/placement decisions and the arch-review record for the resolved blockers.

## Out of scope

- Changing the `/01`–`/09` stage order or count; auto-*authoring* a downstream artifact.
- Weakening any 027 guarantee — `ReBindContentEqual` stays engine-only; the reviewed-no-impact rebind is the only content-touching path and is never auto-applied.
- A TOCTOU CAS token / immutable reviewed snapshot (deferred).
- Folding payload-derived next-actions (`ImpactCommands`/`Scaffold.Cli` data-templated hints) into the model, or adding an `Impact.Cli`/`Scaffold.Cli → Cycle.Core` edge — they stay locally built.
