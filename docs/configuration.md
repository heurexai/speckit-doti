# Doti / `hx` configuration reference

Every setting Doti and `hx` read, grouped by **what it drives**. For each: what it is for, its valid values, and its default. Most settings are derived or auto-configured ‚ÄĒ the small set you actually choose is summarized in the README's [Configuration](../README.md#configuration) section; this document is the complete record.

> Generated from a code-verified inventory (`docs/design/doti-setup-config-inventory.md`). Where a row cites a file, that is the authoritative source.

## How configuration resolves

A value is determined by the first layer that supplies it:

1. **`hx new` / CLI flag** ‚ÄĒ `--name`, `--company`, `--output`, `--agents`, `--profile`, `--force`.
2. **Tracked config file** ‚ÄĒ `.doti/release.json`, `rules/*.json`, `.sentrux/rules.toml`, `GitVersion.yml`, `Directory.Build.props`, `.doti/memory/constitution.md`.
3. **Machine-local file / env var** ‚ÄĒ executable-adjacent `hx.config.json`, `DOTI_RELEASE_ROOT` (not committed).
4. **Derived** ‚ÄĒ computed from name + company + project layout.
5. **Default** ‚ÄĒ the built-in fallback.

The **`Set by`** column below uses: `flag` ¬∑ `.csproj`/`props` ¬∑ `.doti file` ¬∑ `env` ¬∑ `/doti-constitution` ¬∑ **derived** ¬∑ **auto** (set by code, e.g. the first-smoke baseline) ¬∑ **payload-fixed** (ships in the payload; not meant to be edited) ¬∑ **operator-manual** (GitHub/nuget.org-side; `hx` documents the intent but cannot execute it).

## 1. Project identity & package metadata

Drives project/namespace names, the NuGet package, and the assembly/CLI command names.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| `name` | The source name substituted into every project, namespace, file, the package id suffix, the CLI command, and all `release.json` names + the constitution title. | A valid .NET project token (PascalCase, dots allowed), e.g. `Acme.Widget`. | ‚ÄĒ (required) | `flag` `hx new --name` |
| `company` | The `<Company>`, `<Authors>`, the PackageId **prefix**, and the `<Copyright>` holder. | Any string. | `Heurex` | `flag` `hx new --company` |
| `output` | Output directory for the generated repo (`dotnet new -o`). Location only; not substituted into files. | A writable path. | ‚ÄĒ (required for `hx new`) | `flag` `hx new --output` |
| `description` | `<Description>` on the CLI package. | Any string. | A fixed sample sentence ("‚Ä¶a sample agent-first .NET CLI generated by the Heurex scaffold.") | `.csproj` (edit the CLI `.csproj`) |
| `authors` | `<Authors>` independent of company. | Any string. | = `company` | `.csproj` |
| `repositoryUrl` | `<RepositoryUrl>` / `<PackageProjectUrl>` ‚ÄĒ the source link on the NuGet package. | A URL. | none (absent in the template) | `.csproj` |
| `license` | `<PackageLicenseExpression>`. | An SPDX id, e.g. `MIT`, `Apache-2.0`. | `MIT` | `.csproj` |
| `Copyright` | `Copyright ¬© <year> <company>`. | ‚ÄĒ | computed (current year + company) | **derived** |
| `PackageId` | NuGet global-tool id. | ‚ÄĒ | `<company>.<name>` | **derived** |
| `ToolCommandName` | `dotnet tool` command name. | ‚ÄĒ | `<name>` | **derived** |
| `RootNamespace` | CLI project namespace. | ‚ÄĒ | `<name>.Cli` | **derived** |
| Constitution title | The project name in the constitution heading. | ‚ÄĒ | `<name>` (else single `.slnx`/`.sln` base name, else repo dir) | **derived** |

## 2. Versioning

Drives the version the release tags and the package carries (GitVersion-backed).

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| `next-version` | Seeds the version series start; the first `hx release` tags `v0.1.0`. A floor GitVersion ignores once real tags exceed it. | A SemVer, e.g. `0.1.0`. | `0.1.0` | `.doti file` `GitVersion.yml` |
| `workflow` (GitVersion) | The bump model: trunk-based, `main` bumps **patch** by default; `+semver: minor\|major` in a commit body opts up. | A GitVersion workflow id, e.g. `GitHubFlow/v1`. | `GitHubFlow/v1` | `.doti file` `GitVersion.yml` |
| Release intent | Records `major\|minor\|patch` as a `+semver:` signal in the release transition commit, validated against the GitVersion calc. | `major` \| `minor` \| `patch` | blank ‚Üí minor for a feature cycle, patch for a bug-only cycle | `flag` `doti cycle stamp --release-intent` / `hx release --major\|--minor\|--patch` |
| CI GitVersion tool version | Pins CI GitVersion for parity with `hx version calculate`. | A GitVersion release. | `6.7.0` | **payload-fixed** (re-pin in lockstep) |

## 3. Release & local output

Drives `hx release`: what is packed, and where local artifacts are copied. `.doti/release.json` is tracked; `hx.config.json` is machine-local (next to `hx`, not in the repo).

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| `release.json schemaVersion` | Manifest guard; load fails closed unless `1`. | `1` | `1` | **payload-fixed** |
| `release.json productName` | Product/display name in the release identity. | A filesystem-safe name. | `<name>` | `.doti file` (seeded from `--name`) |
| `release.json packageName` | Filesystem-safe name for the local-release subdir. **Not** the NuGet id. | A filesystem-safe name. | `SafeName(productName)` | `.doti file` |
| `release.json publishProject` | The `.csproj` `hx release` packs. | A repo-relative path ending `.csproj`, no `..`, must exist. | `src/<name>.Cli/<name>.Cli.csproj` | `.doti file` |
| `release.json publishedExecutableName` | Assembly/exe name located in the publish output. | A string. | `<name>.Cli` | `.doti file` |
| `release.json executableName` | The final CLI command name the smoke invokes. | A string. | `<name>` | `.doti file` |
| `release.json defaultReleaseRootEnvironmentVariable` | The env-var **name** read for the local release root. | An env-var name. | `DOTI_RELEASE_ROOT` | `.doti file` |
| `hx.config.json schemaVersion` | Guard; throws unless `1`. | `1` | `1` | **payload-fixed** |
| `localReleaseOutput.enabled` | Master switch for the local copy. `false` still runs the tag + pack proofs. | `true` \| `false` | `true` | machine-local `hx.config.json` |
| `localReleaseOutput.directory` | Explicit absolute root; **wins** over the env var. | An absolute path outside the repo, or `null`. | `null` | machine-local `hx.config.json` |
| `localReleaseOutput.environmentVariable` | Machine-local override of the env-var **name**. | An env-var name, or `null`. | `null` ‚Üí release.json default ‚Üí `DOTI_RELEASE_ROOT` | machine-local `hx.config.json` |
| `DOTI_RELEASE_ROOT` (or the configured name) | The actual local directory artifacts copy to. Unset ‚Üí the local copy is skipped (tag + pack still run). | A directory path. | unset | `env` |
| `llmModelRoot` / `HEUREX_LLM_ROOT` | Local LLM root for the **advisory** semantic-drift finder; never gating. | A path, or `null`. | `null` (finder skips) | machine-local `hx.config.json` / `env` |

> **Resolution precedence for the local release root:** `hx.config.json directory` ‚Ėł env var named by `hx.config.json environmentVariable` ‚Ėł `release.json defaultReleaseRootEnvironmentVariable` ‚Ėł `DOTI_RELEASE_ROOT` ‚Ėł skip the copy.

## 6. Architecture & quality gates

Drives the structural/quality gates. Thresholds and layer/namespace contracts are tracked per project; the Sentrux **baseline** is set automatically by the first-smoke run.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| Sentrux baseline | The structural baseline regressions are measured against. | ‚ÄĒ | written by the scaffold first-smoke run (`SentruxBaselineRunner.Save`) | **auto** (never operator; the gate never creates it) |
| `.sentrux/rules.toml` `max_cycles` | Max dependency cycles in production code. | An integer ‚Č• 0 (`0` = acyclic). | `0` | `.doti file` |
| `.sentrux/rules.toml` `max_cc` | Max cyclomatic complexity per function. | A positive integer. | `25` | `.doti file` |
| `.sentrux/rules.toml` `max_fn_lines` | Max function length (the `/08` offender). | A positive integer. | `120` | `.doti file` |
| `.sentrux/rules.toml` `no_god_files` | Flag god-files (high fan-out). | `true` \| `false` | `false` | `.doti file` |
| `.sentrux/rules.toml` `[[layers]]` / `[[boundaries]]` | Layered dependency-direction contract + forbidden edges; project-specific paths. | Path globs + order ints. | rendered for the scaffold layout (`library`(0) ‚Üí `cli`(1)) | **derived** from layout (hand-edit if the layout differs) |
| `rules/sentrux.json` | Gate wiring: `enabled`, `baselinePath`, `signalToleranceBand`, `requiredGrammars`. | bools / ints / lists. | enabled, band `100`, `csharp` | `.doti file` |
| `rules/architecture.json` | Human-readable ArchUnit family contract (the enforcement lives in the `*.Architecture.Tests` project). | Namespaces + family list. | scaffold: 9 families + layers | `.doti file` (+ matching test code) |
| `rules/hygiene.json` | Gitleaks gate: `gitleaksEnabled`, `defaultCommitScope`, `fullScanRequiredForRelease`, `changedFileThreshold`, `excludePaths`, `allowedUrlPrefixes`. | bools / ints / lists. | enabled; changed scope; full @ release; threshold `2000` | `.doti file` |
| `rules/security.json` | `security scan` floor + SCA suppressions. | `auditFloor`: `low`\|`moderate`\|`high`\|`critical`; `suppressions`: list. | `auditFloor=low`, `suppressions=[]` | `.doti file` |
| `Directory.Build.props` `TargetFramework` | The TFM for every project. | A TFM, e.g. `net10.0`. | `net10.0` | `props` (`hx new` does not forward `-f`) |
| `Directory.Build.props` analyzer policy | Build-as-SAST: `TreatWarningsAsErrors`, `AnalysisModeSecurity`, `NuGetAudit*`; CA3xxx/CA5xxx ‚Üí errors. | bools / analyzer modes. | warnings-as-errors; security `All`; audit `all`/`low` | `props` (mostly opinionated) |
| `Nullable` / `ImplicitUsings` / `LangVersion` | Language defaults. | enable/disable / latest. | `enable` / `enable` / `latest` | `props` |
| `architectureTests` | Emit the ArchUnitNET test project. | `true` \| `false` | `true` (always via `hx new`) | **payload-fixed** via `hx new` (raw `dotnet new -a` only) |
| `prerequisites.json` minimums | The .NET SDK / Git minimums + winget ids for `hx prereq`. | versions + winget ids. | .NET SDK ‚Č• `10.0.0`, Git ‚Č• `2.40.0` | **payload-fixed** |
| Gate profile maps | Per-tier `sentrux-verify`/`sentrux-check`/`architecture-test` ‚Üí `enforced\|advisory\|skip` (undeclared ‚Üí enforced). | the three states. | `dotnet-cli`: all enforced; `workflow-only`/`dotnet-lib`: skip | **payload-fixed** (you pick a tier, not edit the map) |

## 7. Constitution (project standards)

Drives what `/03-doti-plan` and `/04-doti-arch-review` evaluate the design against. **¬ß2 is the only operator-authored content**; ¬ß1 is codified and only cited.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| ¬ß2 Domain principles | What the project is + its domain invariants. | Free prose. | `[DOMAIN_PRINCIPLES]` placeholder | `/doti-constitution` |
| ¬ß2 Tech stack | Chosen libs/tools **above** the .NET 10 baseline. | Free prose. | `[TECH_STACK]` placeholder | `/doti-constitution` |
| ¬ß2 Coding style | Conventions beyond ¬ß1 (do not re-declare ¬ß1). | Free prose. | `[CODING_STYLE]` placeholder | `/doti-constitution` |
| ¬ß2 Security & compliance | Posture beyond ¬ß1 hygiene/SAST. | Free prose. | `[SECURITY_COMPLIANCE]` placeholder | `/doti-constitution` |
| ¬ß2 Performance | Performance conventions. | Free prose. | `[PERFORMANCE]` placeholder | `/doti-constitution` |
| ¬ß2 extra headings | Optional free-form headings beyond the five. | Free prose. | none | `/doti-constitution` |
| ¬ß1 inherited invariants | The codified givens (deterministic ownership, thin CLI + `CliResult`, GitVersion, Sentrux/analyzers, the cycle‚Ä¶). | ‚ÄĒ | fixed reference | **payload-fixed** (none are toggle-able by design) |

> Do **not** add a SemVer doc-version, a Sync Impact Report, or re-declare ¬ß1 ‚ÄĒ Doti deliberately omits these.

## 8. Git, branches & CI

Drives the commit/branch/CI shape the release path assumes. Almost all of this is **auto-done by `hx`** or **operator-side** ‚ÄĒ none is a value you tune, but it is here for the record.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| Insurance pre-commit hook | Untracked `.git/hooks/pre-commit` blocking bare commits (refuses to overwrite a foreign hook). | ‚ÄĒ | armed for git repos | **auto** (`doti install-hooks` re-arms per clone) |
| `.gitignore` runtime entries | Ensures `cycle-state.json`, `gate-proof.json`, `sentrux-optimization-log.json`, `templates/` are ignored. | ‚ÄĒ | the four entries | **auto** (additive) |
| Baseline commit | `hx new` stages but does not commit; GitVersion needs a commit and the hook blocks bare commits. | ‚ÄĒ | not made today | **operator-manual** (sanctioned commit) |
| `dev` / `main` two-branch model | The release path assumes it (`cycle stamp --base` defaults to `dev`; release is `dev`‚Üí`main` merge). A fresh `git init` has neither. | branch names. | absent today | **operator-manual** (git) |
| `.github/workflows/{release,ci,dco}.yml` | Tag/pack/publish + CI + DCO automation. The scaffold ships **no** `.github/`. | ‚ÄĒ | absent today | **operator-manual** (copy/author) |
| `main` branch protection | `required_linear_history` **OFF** (dev‚Üímain is a merge commit), required status checks, `enforce_admins`. | GitHub protection settings. | absent | **operator-manual** (GitHub-side; agent is blocked) |

## What you actually set

Most of the table above is derived, auto, or payload-fixed. The genuine operator surface is small ‚ÄĒ see the README [Configuration](../README.md#configuration) section. In short: **name, company, output, agents** at scaffold; **description, repository URL, license, authors, version seed** as package preferences; the **local release directory** (machine-local); **NuGet publish** parameters (if shipping); and the **constitution ¬ß2**. Everything else falls out of those.
