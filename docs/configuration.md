# Doti / `hx` configuration reference

Every setting Doti and `hx` read, grouped by **what it drives**. For each: what it is for, its valid values, and its default. Most settings are derived or auto-configured — the small set you actually choose is summarized in the README's [Configuration](../README.md#configuration) section; this document is the complete record.

> Generated from a code-verified inventory (`docs/design/doti-setup-config-inventory.md`). Where a row cites a file, that is the authoritative source.

## How configuration resolves

A value is determined by the first layer that supplies it:

1. **`--config` / `--interactive`** (`029`) — `hx new` and `hx doti install` take a setup JSON (`--config <doti-setup.json>`) or a wizard (`--interactive`); the resolved choices persist to a tracked `.doti/setup.json`, and `hx doti config show [--json]` shows the effective result with default-vs-custom provenance.
2. **`hx new` / CLI flag** — `--name`, `--company`, `--output`, `--agents`, `--profile`, `--force` (override the matching `--config` field).
3. **Tracked config file** — `.doti/release.json`, `rules/*.json`, `.sentrux/rules.toml`, `GitVersion.yml`, `Directory.Build.props`, `.doti/memory/constitution.md`.
4. **Machine-local file / env var** — executable-adjacent `hx.config.json`, `DOTI_RELEASE_ROOT` (not committed).
5. **Derived** — computed from name + company + project layout.
6. **Default** — the built-in fallback.

The **`Set by`** column below uses: `flag` · `.csproj`/`props` · `.doti file` · `env` · `/doti-constitution` · **derived** · **auto** (set by code, e.g. the first-smoke baseline) · **payload-fixed** (ships in the payload; not meant to be edited) · **operator-manual** (GitHub/nuget.org-side; `hx` documents the intent but cannot execute it).

## Setup config — one file or a wizard (`029`)

`hx new --config <doti-setup.json>` (agent) and `hx new --interactive` (human wizard) capture the operator surface in one pass — mutually exclusive, 1:1 over one schema, validated **before** any file is written, and persisted to a tracked `.doti/setup.json` (repo-portable fields only — a machine-local `release.directory`/`release.enabled` is never written there). The same flags work on `hx doti install` for the doti-layer subset of an existing repo. **`hx doti config show [--json]`** renders the effective config grouped as below, marking each key **default** or **custom** (the resolved model carries `value` / `source` / `default` per key; an absent `.doti/setup.json` shows the all-default view). Operator-only steps (the NuGet OIDC policy, the `NUGET_USER` secret, branch protection, the `v*` tag push) print as an inert **checklist** — never executed.

Minimal `doti-setup.json` — every field is optional except `schemaVersion`; an omitted field takes its documented default:

```jsonc
{
  "schemaVersion": 1,
  "identity": {
    "name": "Acme.Widget",
    "company": "Acme",
    "description": "Acme Widget CLI.",
    "license": "MIT",
    "repositoryUrl": "https://github.com/acme/widget"
  },
  "versioning": { "nextVersion": "0.1.0" },
  "agents": ["codex", "claude"],
  "publish": { "enabled": true, "owner": "acme", "repo": "widget" },
  "constitution": {
    "domainPrinciples": "...", "techStack": "...", "codingStyle": "...",
    "securityCompliance": "...", "performance": "..."
  }
}
```

## 1. Project identity & package metadata

Drives project/namespace names, the NuGet package, and the assembly/CLI command names.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| `name` | The source name substituted into every project, namespace, file, the package id suffix, the CLI command, and all `release.json` names + the constitution title. | A valid .NET project token (PascalCase, dots allowed), e.g. `Acme.Widget`. | — (required) | `flag` `hx new --name` |
| `company` | The `<Company>`, `<Authors>`, the PackageId **prefix**, and the `<Copyright>` holder. | Any string. | `Heurex` | `flag` `hx new --company` |
| `output` | Output directory for the generated repo (`dotnet new -o`). Location only; not substituted into files. | A writable path. | — (required for `hx new`) | `flag` `hx new --output` |
| `description` | `<Description>` on the CLI package. | Any string. | A fixed sample sentence ("…a sample agent-first .NET CLI generated by the Heurex scaffold.") | `.csproj` (edit the CLI `.csproj`) |
| `authors` | `<Authors>` independent of company. | Any string. | = `company` | `.csproj` |
| `repositoryUrl` | `<RepositoryUrl>` / `<PackageProjectUrl>` — the source link on the NuGet package. | A URL. | none (absent in the template) | `.csproj` |
| `license` | `<PackageLicenseExpression>`. | An SPDX id, e.g. `MIT`, `Apache-2.0`. | `MIT` | `.csproj` |
| `Copyright` | `Copyright © <year> <company>`. | — | computed (current year + company) | **derived** |
| `PackageId` | NuGet global-tool id. | — | `<company>.<name>` | **derived** |
| `ToolCommandName` | `dotnet tool` command name. | — | `<name>` | **derived** |
| `RootNamespace` | CLI project namespace. | — | `<name>.Cli` | **derived** |
| Constitution title | The project name in the constitution heading. | — | `<name>` (else single `.slnx`/`.sln` base name, else repo dir) | **derived** |

## 2. Versioning

Drives the version the release tags and the package carries (GitVersion-backed).

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| `next-version` | Seeds the version series start; the first `hx release` tags `v0.1.0`. A floor GitVersion ignores once real tags exceed it. | A SemVer, e.g. `0.1.0`. | `0.1.0` | `.doti file` `GitVersion.yml` |
| `workflow` (GitVersion) | The bump model: trunk-based, `main` bumps **patch** by default; `+semver: minor\|major` in a commit body opts up. | A GitVersion workflow id, e.g. `GitHubFlow/v1`. | `GitHubFlow/v1` | `.doti file` `GitVersion.yml` |
| Release intent | Records `major\|minor\|patch` as a `+semver:` signal in the release transition commit, validated against the GitVersion calc. | `major` \| `minor` \| `patch` | blank → minor for a feature cycle, patch for a bug-only cycle | `flag` `doti cycle stamp --release-intent` / `hx release --major\|--minor\|--patch` |
| CI GitVersion tool version | Pins CI GitVersion for parity with `hx version calculate`. | A GitVersion release. | `6.7.0` | **payload-fixed** (re-pin in lockstep) |

## 3. Release & local output

Drives `hx release`: what is packed, and where local artifacts are copied. `.doti/release.json` is tracked; `hx.config.json` is machine-local (next to `hx`, not in the repo).

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| `release.json schemaVersion` | Manifest guard; load fails closed unless `1`. | `1` | `1` | **payload-fixed** |
| `release.json productName` | Product/display name in the release identity. | A filesystem-safe name. | `<name>` | `.doti file` (seeded from `--name`) |
| `release.json packageName` | Filesystem-safe name for the local-release subdir. **Not** the NuGet id. | A filesystem-safe name. | `SafeName(productName)` | `.doti file` |
| `release.json publishProject` | The `.csproj` `hx release` packs. | A repo-relative path ending `.csproj`, no `..`, must exist. | `src/<name>.Cli/<name>.Cli.csproj` | `.doti file` |
| `release.json publishedExecutableName` | Assembly/exe name located in the publish output. | A string. | `<name>.Cli` | `.doti file` |
| `release.json executableName` | The final CLI command name the smoke invokes. | A string. | `<name>` | `.doti file` |
| `release.json defaultReleaseRootEnvironmentVariable` | The env-var **name** read for the local release root. | An env-var name. | `DOTI_RELEASE_ROOT` | `.doti file` |
| `hx.config.json schemaVersion` | Guard; throws unless `1`. | `1` | `1` | **payload-fixed** |
| `localReleaseOutput.enabled` | Master switch for the local copy. `false` still runs the tag + pack proofs. | `true` \| `false` | `true` | machine-local `hx.config.json` |
| `localReleaseOutput.directory` | Explicit absolute root; **wins** over the env var. | An absolute path outside the repo, or `null`. | `null` | machine-local `hx.config.json` |
| `localReleaseOutput.environmentVariable` | Machine-local override of the env-var **name**. | An env-var name, or `null`. | `null` → release.json default → `DOTI_RELEASE_ROOT` | machine-local `hx.config.json` |
| `DOTI_RELEASE_ROOT` (or the configured name) | The actual local directory artifacts copy to. Unset → the local copy is skipped (tag + pack still run). | A directory path. | unset | `env` |
| `llmModelRoot` / `HEUREX_LLM_ROOT` | Local LLM root for the **advisory** semantic-drift finder; never gating. | A path, or `null`. | `null` (finder skips) | machine-local `hx.config.json` / `env` |

> **Resolution precedence for the local release root:** `hx.config.json directory` ▸ env var named by `hx.config.json environmentVariable` ▸ `release.json defaultReleaseRootEnvironmentVariable` ▸ `DOTI_RELEASE_ROOT` ▸ skip the copy.

## 4. Publishing (NuGet via GitHub Actions)

Drives the automated publish. The workflow parameters are config; the trust-boundary setup is operator-side and cannot be automated.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| NuGet OIDC policy `owner` | The repo owner in the nuget.org Trusted-Publishing policy. | A GitHub owner/org. | — | **operator-manual** (nuget.org) |
| NuGet OIDC policy `repo` | The repo name in the policy. | A repo name. | — | **operator-manual** (nuget.org) |
| NuGet OIDC policy `workflow` | The workflow file the policy trusts. | A workflow filename. | `release.yml` | **operator-manual** (nuget.org) |
| NuGet OIDC policy `environment` | The GitHub Environment gating OIDC issuance; its name **must equal** the policy's Environment field. | An Environment name. | `production` | **operator-manual** (GitHub + nuget.org) |
| `NUGET_USER` | nuget.org username for the OIDC login; no fallback API key. | A nuget.org username. | absent | **operator-manual** (GitHub repo secret) |
| `v*` tag push | Triggers `release.yml` (the publish authorization). | — | — | **operator-manual** (`/09-doti-release` + operator push) |

## 5. Agents & integration

Drives which agent skill trees render and the Doti tier (gate posture).

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| `agents` | Which agent skill trees + root entrypoints render: `claude` → `.claude/skills` + `CLAUDE.md`; `codex` → `.agents/skills` + `AGENTS.md`. | A subset of `claude`, `codex` (others rejected). | `codex,claude` | `flag` `hx new --agents` / `doti install --agents` |
| Tier / `profile` | The gate posture: `dotnet-cli`/`dotnet-cli-heurex` enforce structural gates; `dotnet-lib` adds build/test/security; `workflow-only` skips Sentrux/ArchUnit. | `dotnet-cli` \| `dotnet-cli-heurex` \| `dotnet-lib` \| `workflow-only` | `dotnet-cli` for `hx new`; for `install`: new/empty → `dotnet-cli`, existing non-Doti → `workflow-only`, upgrade → prior | **derived** (no flag today) |

## 6. Architecture & quality gates

Drives the structural/quality gates. Thresholds and layer/namespace contracts are tracked per project; the Sentrux **baseline** is set automatically by the first-smoke run.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| Sentrux baseline | The structural baseline regressions are measured against. | — | written by the scaffold first-smoke run (`SentruxBaselineRunner.Save`) | **auto** (never operator; the gate never creates it) |
| `.sentrux/rules.toml` `max_cycles` | Max dependency cycles in production code. | An integer ≥ 0 (`0` = acyclic). | `0` | `.doti file` |
| `.sentrux/rules.toml` `max_cc` | Max cyclomatic complexity per function. | A positive integer. | `25` | `.doti file` |
| `.sentrux/rules.toml` `max_fn_lines` | Max function length (the `/08` offender). | A positive integer. | `120` | `.doti file` |
| `.sentrux/rules.toml` `no_god_files` | Flag god-files (high fan-out). | `true` \| `false` | `false` | `.doti file` |
| `.sentrux/rules.toml` `[[layers]]` / `[[boundaries]]` | Layered dependency-direction contract + forbidden edges; project-specific paths. | Path globs + order ints. | rendered for the scaffold layout (`library`(0) → `cli`(1)) | **derived** from layout (hand-edit if the layout differs) |
| `rules/sentrux.json` | Gate wiring: `enabled`, `baselinePath`, `signalToleranceBand`, `requiredGrammars`. | bools / ints / lists. | enabled, band `100`, `csharp` | `.doti file` |
| `rules/architecture.json` | Human-readable ArchUnit family contract (the enforcement lives in the `*.Architecture.Tests` project). | Namespaces + family list. | scaffold: 9 families + layers | `.doti file` (+ matching test code) |
| `rules/hygiene.json` | Gitleaks gate: `gitleaksEnabled`, `defaultCommitScope`, `fullScanRequiredForRelease`, `changedFileThreshold`, `excludePaths`, `allowedUrlPrefixes`. | bools / ints / lists. | enabled; changed scope; full @ release; threshold `2000` | `.doti file` |
| `rules/security.json` | `security scan` floor + SCA suppressions. | `auditFloor`: `low`\|`moderate`\|`high`\|`critical`; `suppressions`: list. | `auditFloor=low`, `suppressions=[]` | `.doti file` |
| `Directory.Build.props` `TargetFramework` | The TFM for every project. | A TFM, e.g. `net10.0`. | `net10.0` | `props` (`hx new` does not forward `-f`) |
| `Directory.Build.props` analyzer policy | Build-as-SAST: `TreatWarningsAsErrors`, `AnalysisModeSecurity`, `NuGetAudit*`; CA3xxx/CA5xxx → errors. | bools / analyzer modes. | warnings-as-errors; security `All`; audit `all`/`low` | `props` (mostly opinionated) |
| `Nullable` / `ImplicitUsings` / `LangVersion` | Language defaults. | enable/disable / latest. | `enable` / `enable` / `latest` | `props` |
| `architectureTests` | Emit the ArchUnitNET test project. | `true` \| `false` | `true` (always via `hx new`) | **payload-fixed** via `hx new` (raw `dotnet new -a` only) |
| `prerequisites.json` minimums | The .NET SDK / Git minimums + winget ids for `hx prereq`. | versions + winget ids. | .NET SDK ≥ `10.0.0`, Git ≥ `2.40.0` | **payload-fixed** |
| Gate profile maps | Per-tier `sentrux-verify`/`sentrux-check`/`architecture-test` → `enforced\|advisory\|skip` (undeclared → enforced). | the three states. | `dotnet-cli`: all enforced; `workflow-only`/`dotnet-lib`: skip | **payload-fixed** (you pick a tier, not edit the map) |

## 7. Constitution (project standards)

Drives what `/03-doti-plan` and `/04-doti-arch-review` evaluate the design against. **§2 is the only operator-authored content**; §1 is codified and only cited.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| §2 Domain principles | What the project is + its domain invariants. | Free prose. | `[DOMAIN_PRINCIPLES]` placeholder | `/doti-constitution` |
| §2 Tech stack | Chosen libs/tools **above** the .NET 10 baseline. | Free prose. | `[TECH_STACK]` placeholder | `/doti-constitution` |
| §2 Coding style | Conventions beyond §1 (do not re-declare §1). | Free prose. | `[CODING_STYLE]` placeholder | `/doti-constitution` |
| §2 Security & compliance | Posture beyond §1 hygiene/SAST. | Free prose. | `[SECURITY_COMPLIANCE]` placeholder | `/doti-constitution` |
| §2 Performance | Performance conventions. | Free prose. | `[PERFORMANCE]` placeholder | `/doti-constitution` |
| §2 extra headings | Optional free-form headings beyond the five. | Free prose. | none | `/doti-constitution` |
| §1 inherited invariants | The codified givens (deterministic ownership, thin CLI + `CliResult`, GitVersion, Sentrux/analyzers, the cycle…). | — | fixed reference | **payload-fixed** (none are toggle-able by design) |

> Do **not** add a SemVer doc-version, a Sync Impact Report, or re-declare §1 — Doti deliberately omits these.

## 8. Git, branches & CI

Drives the commit/branch/CI shape the release path assumes. Almost all of this is **auto-done by `hx`** or **operator-side** — none is a value you tune, but it is here for the record.

| Setting | What it is for | Valid values | Default | Set by |
| --- | --- | --- | --- | --- |
| Insurance pre-commit hook | Untracked `.git/hooks/pre-commit` blocking bare commits (refuses to overwrite a foreign hook). | — | armed for git repos | **auto** (`doti install-hooks` re-arms per clone) |
| `.gitignore` runtime entries | Ensures `cycle-state.json`, `gate-proof.json`, `sentrux-optimization-log.json`, `templates/` are ignored. | — | the four entries | **auto** (additive) |
| Baseline commit | `hx new` stages but does not commit; GitVersion needs a commit and the hook blocks bare commits. | — | not made today | **operator-manual** (sanctioned commit) |
| `dev` / `main` two-branch model | The release path assumes it (`cycle stamp --base` defaults to `dev`; release is `dev`→`main` merge). A fresh `git init` has neither. | branch names. | absent today | **operator-manual** (git) |
| `.github/workflows/{release,ci,dco}.yml` | Tag/pack/publish + CI + DCO automation. The scaffold ships **no** `.github/`. | — | absent today | **operator-manual** (copy/author) |
| `main` branch protection | `required_linear_history` **OFF** (dev→main is a merge commit), required status checks, `enforce_admins`. | GitHub protection settings. | absent | **operator-manual** (GitHub-side; agent is blocked) |
| DCO sign-off | Every non-merge PR commit needs `Signed-off-by`; satisfied by an untracked `prepare-commit-msg` hook or `git commit -s`. | — | enforced in CI; hook hand-installed | **operator-manual** (per clone) |

## What you actually set

Most of the table above is derived, auto, or payload-fixed. The genuine operator surface is small — see the README [Configuration](../README.md#configuration) section. In short: **name, company, output, agents** at scaffold; **description, repository URL, license, authors, version seed** as package preferences; the **local release directory** (machine-local); **NuGet publish** parameters (if shipping); and the **constitution §2**. Everything else falls out of those.
