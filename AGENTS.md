# Codex Entry Point

This file is a command-aware-advisory entrypoint for Codex.

Read `.doti/agent-context.md` first. Use local Codex skills under `.agents/skills/doti-*`.

`Hx.Scaffold.Cli new` is the single generation entry point. `gate run --profile <auto|advisory|normal|release>` aggregates the deterministic ladder into one fail-closed `GateProof`. The affected-test planner (`Hx.Impact.Cli plan`) deterministically scopes the gate's normal/advisory test run (project-graph reverse-closure -> covering test projects; fail-closed escalation); release runs the full suite. Versioning is GitVersion-backed (`version calculate` + `version bump`, annotated tag). The security gate (`security scan`) runs package-vulnerability SCA (`dotnet list package --vulnerable`) + reports the build-integrated analyzer SAST status (CA3xxx/CA5xxx as errors — the build is the SAST enforcement point); enforced at release, advisory in dev. The .NET analyzers + Gitleaks are the SAST gate. Vendored tools (Gitleaks/Sentrux/GitVersion) are pinned + hashed (win-x64; large binaries gitignored as an operational vendor step). The gate never creates a Sentrux baseline. Installed skills are rendered from one source — edit `doti/core/skills.json` and re-render; do not hand-edit installed skill files. The doti cycle-state substrate (`doti cycle stamp`/`status`) records diff-bound stage proofs + freshness, and the single-sourced Operator-Question Protocol renders into every skill + the agent context. The enforcing chokepoints are `doti cycle check` (every transitive prerequisite stamped + fresh), `doti cycle commit` (the sole sanctioned commit path — re-verifies a fresh drift-review + task-hash, a fresh passing persisted gate proof, and a clean staged scope, then commits; refuses otherwise), `doti question check` (the operator-question validator), plus an untracked, logic-free insurance pre-commit hook (`doti install-hooks`) that redirects bare commits; `gate run` persists a change-set-bound proof. The workflow is command-enforced at the chokepoints.
